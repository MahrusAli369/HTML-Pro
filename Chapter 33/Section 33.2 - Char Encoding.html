Character Encoding Overview

The charset attribute specifies the character encoding for the HTML document and must match a valid character encoding. Common examples include windows-1252, ISO-8859-2, Shift_JIS, and UTF-8. UTF-8 (Unicode) is the modern gold-standard and should be used for all new projects.

HTML5 Encoding Declaration

<meta charset="UTF-8" />
<meta charset="ISO-8859-1" />

Browsers universally support the meta charset format. However, for strict HTML 4.01 validation rules, use this alternative:

HTML 4.01 Encoding Format

<meta http-equiv="content-type" content="text/html; charset=UTF-8" />
<meta http-equiv="content-type" content="text/html; charset=ISO-8859-1" />
